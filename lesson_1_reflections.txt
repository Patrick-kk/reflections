Q1：你认为，手动选择何时创建提交（像在 Git 中做的那样）与自动保存版本（如 Google Docs 所做的那样）各有何优缺点？

A1：
手动：
优：根据需要自行保存，方便查看历史版本
劣：若发生意外情况会丢失数据

自动：
优：无需手动操作，以免忘记、或发生意外情况而丢失
劣：每次保存都无法确定修改了多少内容，在查看历史版本时不好浏览和区分


Q2：你认为为何一些版本控制系统（例如 Git）允许在一次提交中保存多个文件，而其他系统（例如 Google Docs）单独对待每个文件？

A2：
前者通过“逻辑相关”来作为每一次提交都依据，所以允许涉及多个文件；
后者并不以“逻辑相关”作为依据，仅针对单独一个文件来进行历史版本追踪。

Q3：如何使用命令 git log 和 git diff 来查看文件的历史记录？

A3：
git log 查看所有历史简要记录，包括ID、时间、作者、备注
git log --stat 查看所有历史详细记录，包括ID、时间、作者、备注、修改范围、修改量、影响文件量
git diff ID1 ID2 查看两个ID直接的差异，包括每一行代码变动的内容

Q4：使用版本控制，如何让你更有信心进行有可能会造成破坏的更改？

A4：
可以查看历史版本、比较不同版本之间的差异、可随时回退至特定版本
